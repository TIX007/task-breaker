# Task Breaker Plugin - 快速入门指南

欢迎使用 Task Breaker 插件！本指南将帮助您快速上手并使用插件的核心功能。

## 🚀 30 秒快速开始

### 最简单的使用方式

直接描述您的复杂任务，插件会自动处理：

```
你："帮我研究5个国家的电动汽车市场，生成对比报告"

Task Breaker 会自动：
1. 识别为广度优先任务
2. 创建5个并行研究子任务
3. 同时执行所有研究
4. 合成结果为对比报告
```

**就这么简单！** 无需手动配置，插件会自动完成所有工作。

## 📖 基础概念

### 什么是任务分解？

任务分解是将一个大的、复杂的任务拆分成多个小的、可管理的子任务的过程。

**为什么需要任务分解？**

- ✅ 降低复杂度：大任务 → 多个简单小任务
- ✅ 提高效率：并行执行 → 显著减少总时间
- ✅ 便于管理：清晰的子任务 → 更好的进度跟踪
- ✅ 保证质量：每个子任务都有明确的成功标准

### Task Breaker 的核心价值

| 传统方式 | 使用 Task Breaker |
|---------|-----------------|
| 顺序执行 | 并行执行 |
| 手动管理依赖 | 自动依赖管理 |
| 难以跟踪进度 | 实时进度监控 |
| 容易遗漏任务 | 系统化分解 |
| 时间长、效率低 | 时间短、效率高 |

## 🎯 三种任务类型

Task Breaker 会自动识别任务类型并采用最佳策略：

### 1. 深度优先任务（Depth-First）

**特征**：从多个角度分析同一问题

**示例**：
```
"分析 AI 对医疗行业的影响"
```

**自动分解为**：
- 角度 1：监管和政策影响
- 角度 2：临床应用和效果
- 角度 3：经济影响和成本
- 角度 4：技术挑战和限制
- 合成：综合分析报告

### 2. 广度优先任务（Breadth-First）

**特征**：分解为多个独立的子问题

**示例**：
```
"比较5个国家的税收系统"
```

**自动分解为**：
- 子任务 1：研究国家 A
- 子任务 2：研究国家 B
- 子任务 3：研究国家 C
- 子任务 4：研究国家 D
- 子任务 5：研究国家 E
- 合成：对比分析报告

### 3. 直接型任务（Straightforward）

**特征**：有明确依赖关系的顺序步骤

**示例**：
```
"处理这个 Excel 文件并生成报告"
```

**自动分解为**：
- 步骤 1：读取和验证数据
- 步骤 2：清洗和转换数据
- 步骤 3：分析和生成洞察
- 步骤 4：生成报告并保存

## 💡 实用示例

### 示例 1：市场研究

**你的请求**：
```
研究2024年全球电商市场的趋势，包括移动购物、社交媒体营销、
个性化推荐、跨境支付和物流创新
```

**Task Breaker 处理**：
```
识别为：广度优先任务
创建子任务：
  1. 研究移动购物趋势
  2. 研究社交媒体营销
  3. 研究个性化推荐技术
  4. 研究跨境支付方案
  5. 研究物流创新
  6. 合成综合报告
执行方式：1-5 并行执行，最后执行 6
预计时间：30-40 分钟（顺序需 2-3 小时）
```

### 示例 2：软件开发

**你的请求**：
```
帮我规划开发一个博客系统的任务
```

**Task Breaker 处理**：
```
识别为：混合型任务
创建子任务：
  1. 设计数据库架构（基础）
  2. 开发后端 API（依赖 1）
  3. 开发前端界面（与 2 并行）
  4. 实现用户认证（与 2,3 并行）
  5. 前后端集成（依赖 2,3,4）
  6. 测试和优化（依赖 5）
执行方式：1 → [2,3,4 并行] → 5 → 6
预计时间：总计时间明显小于顺序执行
```

### 示例 3：数据分析

**你的请求**：
```
分析销售数据，找出关键趋势和机会
```

**Task Breaker 处理**：
```
识别为：混合型任务
创建子任务：
  1. 数据清洗和准备（基础）
  2. 销售趋势分析（依赖 1）
  3. 客户行为分析（依赖 1，与 2 并行）
  4. 产品性能分析（依赖 1，与 2,3 并行）
  5. 地域分布分析（依赖 1，与 2,3,4 并行）
  6. 生成洞察报告和可视化（依赖 2,3,4,5）
执行方式：1 → [2,3,4,5 并行] → 6
预计时间：数据并行分析，效率显著提升
```

## 🛠️ 高级用法

### 自定义分解粒度

如果默认分解的子任务过多或过少，可以添加具体要求：

```
"帮我研究3个国家的市场情况，不要超过4个子任务"
```

这样会创建更粗粒度的子任务。

### 指定优先级

可以明确指出某些方面更重要：

```
"重点研究中国市场，其他3个国家可以简略一些"
```

这样会分配更多资源给高优先级任务。

### 添加特定要求

可以添加任何具体要求：

```
"研究5个国家的能源政策，特别关注可再生能源的
补贴政策和投资情况，需要包括最近3年的数据"
```

具体要求会被 incorporated 到每个相关子任务中。

### Token 效率

虽然并行执行会创建更多子任务，但：

- ✅ 总体 token 消耗降低（因为并行更高效）
- ✅ 避免重复工作
- ✅ 结果质量更高（专注的子任务）

## ⚠️ 常见问题

### Q1: Task Breaker 什么时候会自动激活？

**A**: 当您的请求包含以下关键词时自动激活：
- 中文："拆分任务"、"分解任务"、"子任务"、"任务拆解"
- 英文："break down task"、"split task"、"subtask"、"task decomposition"

### Q2: 我可以手动触发 Task Breaker 吗？

**A**: 可以！直接说：
```
"使用 Task Breaker 来规划这个任务：..."
```

或者：
```
"帮我分解一下这个任务：..."
```

### Q3: 如果任务太简单会怎样？

**A**: Task Breaker 会智能判断，对于简单任务（< 5 分钟可完成）会建议直接执行，避免不必要的分解。

### Q4: 可以控制子任务数量吗？

**A**: 可以！添加明确要求：
```
"最多3个子任务"
"详细分解，至少8个子任务"
```

### Q5: 并行执行会影响结果质量吗？

**A**: 不会！相反：
- 每个子任务更专注
- 更容易发现问题
- 更好的质量控制
- 系统化的合成流程

### Q6: 如果某个子任务失败了怎么办？

**A**: Task Breaker 有完整的错误处理机制：
- 重试机制
- 范围调整
- 替代方案
- 关键任务失败会及时通知您

## 🎓 最佳实践

### ✅ 推荐做法

1. **清晰描述目标**
   ```
   好："研究5个国家的可再生能源政策，生成对比报告"
   不好："研究可再生能源"
   ```

2. **提供足够上下文**
   ```
   好："分析销售数据，重点看Q3和Q4的表现，对比去年同期"
   不好："分析销售数据"
   ```

3. **明确输出格式**
   ```
   好："生成包含图表和表格的PDF报告"
   不好："生成报告"
   ```

4. **设定时间约束**
   ```
   好："在1小时内完成研究"
   不好："做研究"
   ```

### ❌ 避免做法

1. **过度复杂的请求**
   ```
   ❌："研究所有国家的所有政策"
   ```
   建议限制范围和数量

2. **模糊的需求**
   ```
   ❌："做一个好点的分析"
   ```
   应该明确具体要求

3. **相互矛盾的要求**
   ```
   ❌："详细研究10个国家，但只要5分钟"
   ```
   时间和范围要合理
